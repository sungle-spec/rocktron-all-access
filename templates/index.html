Write All — pacing (locked)

Pacing is hard-locked to the All Access manual's literal "1 byte every 15 ms" spec, confirmed working on real hardware. Any other values (chunked sends, longer byte delays, or any inter-frame pause) caused the device to commit early during testing and corrupted the rig — so these are read-only.

Chunk size1 byte · byte-by-byte
Byte delay15 ms · matches device input rate (~65 Hz)
Inter-frame pause0 ms · continuous stream — never insert silence

Full dump ETA: ~11 minutes (43 647 bytes × 15 ms). Don't switch browser tabs or let the laptop sleep during Write All — keep tail -f /tmp/aa-server.log | grep WRITE_ALL open in a terminal for live progress visible to the user.

Songs — 150 banks of presets (SONG/SET P2)

Each song = 1 bank. With Bank Size = 5, only slots 1–5 are active on the device; the editor exposes all 10 slots since they're stored regardless.

Sets — 10 sets × 50 banks → song (SONG/SET P3)

Each set selects which song plays in each of the 50 banks.

PC Map — incoming Program Change → preset (MIDI P6)

When the device receives PCn on its MIDI Receive Channel and Program Change Status = MAP, it recalls the mapped preset. Default mapping is identity (PC1→PR1, PC2→PR2, …).

Rocktron All Access — Web Editor & Virtual Foot Controller

Browser-based editor for the Rocktron All Access MIDI foot controller (1994). Reverse-engineered from a single bulk dump and 6 targeted capture/diff dumps. Manufacturer ID 00 00 29, Device ID 08; bulk dump = 145 sysex frames totalling 43 647 bytes.

What you can edit

  • Preset name — 13 chars, A-Z 0-9 (the device's . renders as space).
  • 16 PC slots per preset — value (0-127) + active/OFF flag, one per MIDI channel.
  • Instant-access on/off bitmap — all 15 SWs (SW1-15), saved per preset via CTR STORE.
  • Custom MIDI commands — 4 of the 5 CMD slots per preset (CMD5 location TBD).
  • SysEx string — 30 bytes per preset + ON/OFF toggle.
  • Songs — all 150 songs, 10 preset slots each.
  • Sets — all 10 sets, 50 banks → song mapping each.
  • PC Map — 128 incoming PC → preset slots.
  • Channel / switch / pedal names — 4-char custom names spliced into the name block.
  • Globals — Operating Mode, Bank Size, Bank Style, MIDI Receive Channel (incl. OMNI), Starting Preset, Remote Title, Program Change Status.

What's read-only / partial

  • Switch type (LATCH/MOM/HOLD per IA switch) — encoding decoded but slot indexing scheme is stack-style and not fully mapped, so it's hidden from the UI for safety.
  • 5th custom MIDI command (CMD5) — its byte position would overlap the SysEx region; only CMD1-4 are exposed.
  • Per-preset PER-PR override block — partial; UI not wired yet.
  • MIDI filter mask — single bit-edit confirmed, full bitmap layout not yet decoded.

Workflow

  1. Connect the All Access via USB-MIDI (7-pin DIN works fine — pins 1/3 carry MIDI, 6/7 carry phantom power).
  2. Pick your IN and OUT ports from the header dropdowns. LEDs go green when open.
  3. Read All (cloud-down icon) — arms the editor; trigger SYSX → DUMP → CTR STORE on the device.
  4. Edit any field in any tab. Each section's Save button splices your changes into the loaded dump bytes (non-destructive — every other byte preserved).
  5. Save .syx (floppy icon) — downloads the current in-memory dump to your ~/Downloads as a backup.
  6. Write All (yellow upload icon) — sends the whole dump to the device. Put the unit on SYSX → LOAD first. Throttled to ~65 Hz per the manual to avoid Buffer Overflow.
